Alert emails:
- Sent separately when any of the following are detected:
  - Camera unreachable
  - Camera reachable but no recording from yesterday (gap alert)
  - Camera NTP not synced to Pi (wrong server, wrong source, or offset >= 5s)
  - Pull exit code non-zero
  - SSD usage >= ALERT_DISK_THRESHOLD (default 80%)
  - Pi CPU temp >= ALERT_TEMP_THRESHOLD (default 70 C)
- Subject line starts with [ALERT] for easy Gmail filtering
- ALERT_EMAIL_TO can be set to route alerts to a different address

KNOWN GOTCHAS
-------------
- Axis API username is root, not admin — using admin returns 401
- Axis API times are UTC; filenames and logs preserve UTC
- Active recordings must never be exported; only recordingstatus="completed"
- ffprobe checks must be run one file at a time, not with a bare wildcard
- SSD days-remaining estimate requires >1 day of file history; shows on day 2+
- SD days-remaining estimate stabilizes after more recordings accumulate
- Read-only root filesystem (raspi-config overlayfs) broke networking — do not enable
- GPS cold start takes 5-15 minutes; system falls back to NTP automatically
- First-day cellular setup uses significant data (~35 MB); steady state is ~2 MB/day
- Disable wifi1 on RUT241 before field deployment to prevent data waste
