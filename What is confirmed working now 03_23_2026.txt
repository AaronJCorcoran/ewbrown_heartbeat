Report includes (Quick Check summary + detail):
- Pi uptime, CPU temperature
- SSD used/free/% with estimated days remaining (based on write rate)
- Per camera: reachable, completed/active recording count
- Per camera: yesterday's total recording duration (sum of completed recordings
  that started on the previous calendar day in local time)
- Per camera: SD card used/free/% with estimated days remaining
- Per camera: oldest recording on SD card (indicates buffer depth before overwrite)
- Per camera: live JPEG snapshot attached to email
- Pull service result and exit code
- Latest 3 files on SSD per camera
- Recent pull log errors

Alert emails:
- Sent separately when any of the following are detected:
  - Camera unreachable
  - Camera reachable but no recording from yesterday (gap alert)
  - Pull exit code non-zero
  - SSD usage >= ALERT_DISK_THRESHOLD (default 80%)
  - Pi CPU temp >= ALERT_TEMP_THRESHOLD (default 70 C)
- Subject line starts with [ALERT] for easy Gmail filtering
- ALERT_EMAIL_TO can be set to route alerts to a different address
